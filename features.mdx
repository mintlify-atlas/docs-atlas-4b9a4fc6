## User Management

### Authentication & Authorization

RegistroX implements a secure authentication system with:

- **Email/Password Login** - Standard authentication with form validation
- **User Registration** - Create new accounts with role assignment
- **OTP Verification** - Two-factor authentication via one-time codes
- **Session Persistence** - Automatic login using DataStore
- **Role-Based Access** - Different interfaces for users and workers

```kotlin
// Login with credentials
loginViewModel.login(email, password)

// Verify OTP code
loginViewModel.verifyOtp(code)
```

### Role System

Two distinct user roles provide tailored experiences:

| Role | Access Level | Features |
|------|-------------|----------|
| **USUARIO** | Standard User | Browse and purchase tickets, view purchase history |
| **TRABAJADOR** | Event Worker | Access scanner interface, validate tickets at entry |

## Ticket Management

### Event Browsing

Users can browse available tickets with comprehensive details:

- Event title and location
- Ticket pricing and availability
- Real-time status updates
- Search and filter capabilities

### Purchase Flow

Complete checkout process includes:

1. Browse available tickets
2. Add tickets to shopping cart
3. Review cart with quantities
4. Complete purchase
5. Generate unique QR codes
6. Store tickets locally and remotely

### Ticket States

Tickets progress through multiple states:

- **disponible** - Available for purchase
- **vendido** - Sold but not yet used
- **usado** - Validated and used at event
- **cancelado** - Cancelled or refunded

## QR Code System

### Code Generation

Each purchased ticket receives a unique QR code:

```kotlin
val codigoQR = "TICKET-${usuarioId}-${entrada.id}-${System.currentTimeMillis()}"
```

Security features:
- Timestamp-based uniqueness
- User ID association
- Ticket ID reference
- Server-side validation

### Scanning & Validation

Workers can scan and validate tickets:

- Camera-based QR scanning using ZXing
- Real-time validation against backend
- Visual feedback (success/error states)
- Offline fallback with manual entry

## Data Synchronization

### Real-Time Updates

RegistroX maintains data consistency through:

- Automatic API synchronization
- Local caching with DataStore
- Pull-to-refresh capabilities
- Background updates

### Offline Support

Key features work offline:
- View purchased tickets
- Access profile information
- Browse cached ticket data

Data syncs automatically when connection is restored.

## User Interface

### Navigation System

Modern navigation patterns:

- Bottom navigation bar with role-based items
- Conditional routing based on authentication
- Deep linking support
- Smooth screen transitions

### Top Bar Features

Dynamic top bar adapts to context:

- **Home** - Shopping cart icon with badge count
- **Worker** - Logout button for session management
- **Default** - App title and back navigation

## Permissions

RegistroX requires specific permissions:

- **CAMERA** - QR code scanning
- **INTERNET** - API communication
- **READ_EXTERNAL_STORAGE** - Profile image upload
- **ACCESS_NETWORK_STATE** - Network status monitoring
- **POST_NOTIFICATIONS** - System notifications
