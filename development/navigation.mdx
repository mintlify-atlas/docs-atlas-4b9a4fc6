## Navigation Architecture

RegistroX uses Jetpack Compose Navigation to handle app navigation. The navigation system is built on three core components:

1. **Routes** - String constants defining navigation destinations
2. **NavGraph** - Composable navigation graph connecting routes to screens
3. **BottomNavItem** - Bottom navigation item definitions

## Routes Definition

All navigation routes are centralized in a single object:

```kotlin Routes.kt
object Routes {
    const val OTP = "otp"
    const val LOGIN = "login"
    const val HOME = "home"
    const val ENTRADAS = "entradas"
    const val REGISTER = "register"
    const val PROFILE = "profile"
    const val DETALLE = "detalle/{entradaId}"
    const val TRABAJADOR = "trabajador"
    const val COMPRAS = "compras"
}
```

<Note>
Routes are defined as constants to ensure type safety and prevent typos when navigating between screens.
</Note>

## Navigation Graph

The NavGraph connects routes to their corresponding screen composables:

<Accordion title="NavGraph Implementation">
```kotlin NavGraph.kt
@Composable
fun NavGraph(
    navController: NavHostController,
    carritoViewModel: CarritoViewModel,
    loginViewModel: LoginViewModel,
    registerViewModel: RegisterViewModel,
    profileViewModel: ProfileViewModel,
    entradasApiViewModel: EntradasApiViewModel,
    comprasViewModel: ComprasViewModel,
    modifier: Modifier = Modifier
) {
    val user by loginViewModel.user.collectAsStateWithLifecycle()
    
    // Dynamic start destination based on auth state
    val startDestination = if (user == null) Routes.LOGIN else Routes.HOME
    
    NavHost(
        navController = navController,
        startDestination = startDestination,
        modifier = modifier
    ) {
        composable(Routes.LOGIN) {
            LoginScreen(navController, loginViewModel)
        }
        
        composable(Routes.OTP) {
            OtpScreen(
                navController = navController,
                viewModel = loginViewModel,
                onOtpVerified = {
                    navController.navigate(Routes.HOME) {
                        popUpTo(Routes.LOGIN) { inclusive = true }
                    }
                }
            )
        }
        
        composable(Routes.HOME) {
            HomeScreen(navController, carritoViewModel)
        }
        
        composable(Routes.TRABAJADOR) {
            val current = user
            if (current?.role == Role.TRABAJADOR) {
                HomeTrabajadorScreen(
                    onBackClick = { navController.navigate(Routes.HOME) },
                    carritoViewModel = carritoViewModel
                )
            } else {
                LaunchedEffect(Unit) {
                    navController.navigate(Routes.HOME) {
                        popUpTo(Routes.TRABAJADOR) { inclusive = true }
                    }
                }
            }
        }
        
        composable(Routes.REGISTER) {
            RegisterScreen(navController, registerViewModel)
        }
        
        composable(Routes.ENTRADAS) {
            EntradasScreen(
                navController = navController,
                carritoViewModel = carritoViewModel
            )
        }
        
        composable(Routes.PROFILE) {
            val current = user
            if (current != null) {
                ProfileScreen(
                    user = current,
                    loginViewModel = loginViewModel,
                    navController = navController,
                    profileViewModel = profileViewModel
                )
            } else {
                LaunchedEffect(Unit) {
                    navController.navigate(Routes.LOGIN) {
                        popUpTo(Routes.PROFILE) { inclusive = true }
                    }
                }
            }
        }
        
        // Route with path parameter
        composable("${Routes.DETALLE}/{codigoQR}") { backStackEntry ->
            val codigoQR = backStackEntry.arguments?.getString("codigoQR") ?: ""
            DetalleEntradaScreen(navController, codigoQR)
        }
        
        composable(Routes.COMPRAS) {
            ComprasScreen(navController, comprasViewModel)
        }
    }
}
```
</Accordion>

## Navigation Patterns

### Basic Navigation

Navigate to a destination:

```kotlin
navController.navigate(Routes.HOME)
```

### Navigation with Arguments

Pass arguments in the route path:

```kotlin
// Navigate to detail with parameter
navController.navigate("detalle/${entrada.codigoQR}")

// Extract parameter in destination
composable("detalle/{codigoQR}") { backStackEntry ->
    val codigoQR = backStackEntry.arguments?.getString("codigoQR") ?: ""
    DetalleEntradaScreen(navController, codigoQR)
}
```

### Pop Back Stack

Navigate back and remove destinations:

```kotlin
// Simple back navigation
navController.popBackStack()

// Navigate and clear back stack
navController.navigate(Routes.HOME) {
    popUpTo(Routes.LOGIN) { inclusive = true }
}
```

### Single Top Launch

Prevent multiple instances of the same destination:

```kotlin
navController.navigate(Routes.HOME) {
    launchSingleTop = true
    restoreState = true
}
```

## Conditional Navigation

<Accordion title="Role-Based Navigation">
The app implements role-based access control in navigation:

```kotlin
composable(Routes.TRABAJADOR) {
    val current = user
    if (current?.role == Role.TRABAJADOR) {
        HomeTrabajadorScreen(
            onBackClick = { navController.navigate(Routes.HOME) },
            carritoViewModel = carritoViewModel
        )
    } else {
        // Redirect non-workers
        LaunchedEffect(Unit) {
            navController.navigate(Routes.HOME) {
                popUpTo(Routes.TRABAJADOR) { inclusive = true }
            }
        }
    }
}
```
</Accordion>

## Bottom Navigation Integration

The bottom bar integrates with NavController to handle tab navigation:

```kotlin MainActivity.kt
val bottomItems = listOf(
    BottomNavItem.Home, 
    BottomNavItem.QR, 
    BottomNavItem.Profile
) + if (user?.role == Role.TRABAJADOR) 
    listOf(BottomNavItem.Scan) 
  else emptyList()

BottomBar(
    navController = navController,
    items = bottomItems,
    userRole = user?.role
)
```

### Bottom Bar Navigation Logic

```kotlin BottomBar.kt
NavigationBarItem(
    selected = currentRoute == item.route,
    onClick = {
        if (currentRoute != item.route) {
            navController.navigate(item.route) {
                // Pop to HOME and save state
                popUpTo(Routes.HOME) { saveState = true }
                launchSingleTop = true
                restoreState = true
            }
        }
    }
)
```

## Conditional UI Elements

The app conditionally shows/hides navigation elements based on route:

```kotlin MainActivity.kt
val navBackStackEntry by navController.currentBackStackEntryAsState()
val currentRoute = navBackStackEntry?.destination?.route

val hideTopBar = currentRoute in listOf(
    Routes.LOGIN, 
    Routes.REGISTER, 
    Routes.DETALLE, 
    Routes.OTP
)

val hideBottomBar = currentRoute in listOf(
    Routes.LOGIN, 
    Routes.REGISTER, 
    Routes.DETALLE, 
    Routes.OTP
)

Scaffold(
    topBar = {
        if (!hideTopBar && user != null) {
            // Show appropriate top bar
        }
    },
    bottomBar = {
        if (!hideBottomBar && user != null) {
            BottomBar(...)
        }
    }
) { padding ->
    NavGraph(...)
}
```

## Deep Linking

To add deep link support, define deep links in the navigation graph:

```kotlin
composable(
    route = "detalle/{codigoQR}",
    deepLinks = listOf(
        navDeepLink {
            uriPattern = "registrox://detalle/{codigoQR}"
        }
    )
) { backStackEntry ->
    val codigoQR = backStackEntry.arguments?.getString("codigoQR") ?: ""
    DetalleEntradaScreen(navController, codigoQR)
}
```

Add intent filter to AndroidManifest.xml:

```xml AndroidManifest.xml
<activity android:name=".MainActivity">
    <intent-filter>
        <action android:name="android.intent.action.VIEW" />
        <category android:name="android.intent.category.DEFAULT" />
        <category android:name="android.intent.category.BROWSABLE" />
        <data android:scheme="registrox" />
    </intent-filter>
</activity>
```

<Note>
Deep linking allows external apps, notifications, or QR codes to navigate directly to specific screens within RegistroX.
</Note>

## Navigation Best Practices

1. **Centralize routes** - Use the Routes object to avoid hardcoded strings
2. **Clear back stack** - Use `popUpTo` when navigating to authenticated/unauthenticated flows
3. **Single top launch** - Prevent duplicate destinations in the stack
4. **Save state** - Use `saveState` and `restoreState` for bottom navigation
5. **Type safety** - Consider using Navigation Compose type-safe arguments for complex parameters
6. **Conditional navigation** - Guard routes based on authentication and role
