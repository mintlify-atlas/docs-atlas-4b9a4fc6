## Overview

RegistroX provides comprehensive profile management features allowing users to personalize their accounts with profile images and manage their account settings including logout functionality.

<Note>
Profile operations require authentication and internet connectivity for image uploads and synchronization.
</Note>

## Profile Implementation

The profile screen is implemented with role-based display and image management:

```kotlin ProfileScreen.kt:27
@Composable
fun ProfileScreen(
    user: User,
    loginViewModel: LoginViewModel,
    navController: NavController,
    profileViewModel: ProfileViewModel
) {
    val imageUrl = profileViewModel.imageUriState.value

    LaunchedEffect(Unit) {
        profileViewModel.sincronizarImagenDesdeApi(user.id)
    }

    val pickImageLauncher = rememberLauncherForActivityResult(
        contract = ActivityResultContracts.GetContent(),
        onResult = { uri: Uri? ->
            uri?.let { profileViewModel.subirImagen(user.id, it) }
        }
    )

    val perfilTexto = when (user.role) {
        Role.TRABAJADOR -> "Perfil de trabajador"
        else -> "Perfil de usuario"
    }
}
```

### Upload Implementation

The `ProfileViewModel` handles image upload with the backend API:

```kotlin ProfileViewModel.kt:40
fun subirImagen(usuarioId: Long, localUri: Uri) {
    viewModelScope.launch {
        val context = getApplication<Application>().applicationContext
        if (!NetworkUtils.isNetworkAvailable(context)) return@launch

        try {
            val multipart = uriToMultipart(context, localUri, "file")

            val response = RetrofitInstance.api.subirImagenMultipart(
                multipart,
                usuarioId
            )

            if (response.isSuccessful) {
                val imageUrl = response.body()?.imageUrl ?: ""
                dataStore.saveImageUri(imageUrl)
                imageUriState.value = imageUrl
            }

        } catch (_: Exception) {
        }
    }
}
```

## Image Synchronization

Profile images are automatically synchronized from the API when the profile screen loads.

### Sync from API

```kotlin ProfileViewModel.kt:64
fun sincronizarImagenDesdeApi(usuarioId: Long) {
    viewModelScope.launch {
        val context = getApplication<Application>().applicationContext
        if (!NetworkUtils.isNetworkAvailable(context)) return@launch

        try {
            val url = authRepository.getImagenPorUsuarioId(usuarioId)
            if (url != null) {
                dataStore.saveImageUri(url)
                imageUriState.value = url
            }
        } catch (_: Exception) {
        }
    }
}
```

### Local Persistence

Images are cached locally using DataStore for offline access:

```kotlin ProfileViewModel.kt:24

}
```

## Error Handling

The profile system gracefully handles various scenarios:

- **No Internet**: Upload operations are skipped silently
- **Upload Failure**: User can retry the upload
- **Missing Image**: Default icon is displayed
- **Sync Failure**: Cached image is used if available
