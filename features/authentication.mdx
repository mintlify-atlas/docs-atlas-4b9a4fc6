## Overview

RegistroX implements a secure authentication system with email-based login, user registration with validation, and OTP (One-Time Password) verification for enhanced security.

<Note>
All authentication flows require internet connectivity and are protected by the `InternetGuard` component.
</Note>

## Authentication Workflows

<Steps>
  <Step title="User Registration">
    New users create an account with email and password validation
  </Step>
  <Step title="Login">
    Existing users authenticate with their credentials
  </Step>
  <Step title="OTP Verification">
    Users verify their identity with a 4-digit code sent via notification
  </Step>
</Steps>

### Implementation

The login screen is implemented in `LoginScreen.kt` and uses the `LoginViewModel` for state management:

```kotlin LoginScreen.kt:22
@Composable
fun LoginScreen(navController: NavController, viewModel: LoginViewModel) {
    val user by viewModel.user.collectAsStateWithLifecycle()
    val formState by viewModel.formState.collectAsStateWithLifecycle()
    val isLoading by viewModel.isLoading.collectAsStateWithLifecycle()

    LaunchedEffect(user) {
        if (user != null && viewModel.justLoggedIn) {
            navController.navigate(Routes.OTP) {
                popUpTo(Routes.LOGIN) { inclusive = true }
            }
            viewModel.justLoggedIn = false
        }
    }
}
```

### Login ViewModel

The `LoginViewModel` manages authentication state and communicates with the backend:

```kotlin LoginViewModel.kt:60
fun login() {
    viewModelScope.launch {
        _isLoading.value = true
        delay(800)

        try {
            val email = _formState.value.email.trim()
            val password = _formState.value.password

            val usuarioAPI = authRepository.login(email, password)

            if (usuarioAPI != null) {
                val roleEnum = 
                    if (usuarioAPI.rol.id?.toInt() == 1) Role.TRABAJADOR 
                    else Role.USUARIO

                justLoggedIn = true

                _user.value = User(
                    id = usuarioAPI.id ?: 0L,
                    email = usuarioAPI.email,
                    role = roleEnum
                )

                authDataStore.saveUser(usuarioAPI.email, roleEnum.name)
            } else {
                _formState.update { it.copy(loginError = "Credenciales incorrectas") }
            }
        } catch (e: Exception) {
            _formState.update { it.copy(loginError = "Error: ${e.localizedMessage}") }
        }

        _isLoading.value = false
    }
}
```

## Registration Screen

New users can create accounts with comprehensive validation for email and password fields.

### Validation Rules

<Accordion title="Email Validation">
  - Must not be blank
  - Must match email format: `username@domain.extension`
  - Validated using regex pattern: `^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,}$`
</Accordion>

<Accordion title="Password Validation">
  - Minimum 6 characters
  - Must contain at least one uppercase letter
  - Must contain at least one number
  - Validated using regex: `^(?=.*[A-Z])(?=.*\\d).{6,}$`
</Accordion>

<Accordion title="Confirm Password">
  - Must match the password field
  - Real-time validation as user types
</Accordion>

### Role Assignment

Users are automatically assigned roles based on their email domain:

```kotlin RegisterViewModel.kt:125
val roleEnum = if (email.endsWith("@registrox.cl"))
```

- Emails ending with `@registrox.cl` receive the `TRABAJADOR` (Worker) role
- All other emails receive the `USUARIO` (User) role

### OTP Input Component

The OTP screen features a custom input field that displays four separate boxes for each digit:

```kotlin OtpScreen.kt:151
@Composable
fun OtpInputField(code: String, onCodeChange: (String) -> Unit) {
    val focusRequester = remember { FocusRequester() }

    TextField(
        value = code,
        onValueChange = { value ->
            if (value.length <= 4 && value.all { it.isDigit() }) {
                onCodeChange(value)
            }
        },
        keyboardOptions = KeyboardOptions(keyboardType = KeyboardType.Number)
    )

    val chars = List(4) { i -> code.getOrNull(i)?.toString() ?: "" }

    Row(horizontalArrangement = Arrangement.spacedBy(12.dp)) {
        chars.forEach { digit ->
            Surface(
                shape = RoundedCornerShape(8.dp),
                border = BorderStroke(1.dp, MaterialTheme.colorScheme.primary),
                modifier = Modifier.width(56.dp).height(56.dp)
            ) {
                Box(contentAlignment = Alignment.Center) {
                    Text(text = digit, fontSize = 20.sp)
                }
            }
        }
    }
}
```

### Notification Handling

The app requests notification permissions (Android 13+) and sends the OTP via notification:

```kotlin OtpScreen.kt:221
fun sendOtpNotification(context: Context, code: String) {
    val channelId = "otp_channel"
    val manager = NotificationManagerCompat.from(context)

    if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.O) {
        manager.createNotificationChannel(
            NotificationChannel(
                channelId,
                "OTP Notifications",
                NotificationManager.IMPORTANCE_HIGH
            )
        )
    }

    val notification = NotificationCompat.Builder(context, channelId)
        .setSmallIcon(android.R.drawable.ic_lock_idle_lock)
        .setContentTitle("OTP Code")
        .setContentText("Tu codigo es: $code")
        .setPriority(NotificationCompat.PRIORITY_HIGH)
        .build()

    manager.notify(1, notification)
}
```

The authentication system provides clear error messages for various scenarios:

- **Invalid Credentials**: "Credenciales incorrectas"
- **Network Error**: "Error de red" or "Sin conexion a internet"
- **Validation Errors**: Field-specific error messages
- **OTP Mismatch**: "Código incorrecto"
