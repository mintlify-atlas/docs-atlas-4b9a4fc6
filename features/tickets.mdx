<Note>
All ticket operations require an active internet connection and authenticated user session.
</Note>

## Loading Tickets

The `EntradasApiViewModel` manages ticket data retrieval and state:

```kotlin EntradasApiViewModel.kt:21
fun cargarEntradas(hasInternet: Boolean) {
    viewModelScope.launch {
        if (!hasInternet) {
            _mensaje.value = "Sin conexion a internet"
            return@launch
        }

        try {
            val response = repository.obtenerEntradas()
            if (response.isSuccessful) {
                _entradas.value = response.body() ?: emptyList()
            } else {
                _mensaje.value = "Error: ${response.code()}"
            }
        } catch (e: Exception) {
            _mensaje.value = "Error de red"
        }
    }
}
```

## QR Code System

Each purchased ticket generates a unique QR code for event entry validation.

### QR Code Generation

QR codes are uniquely generated for each ticket purchase:

```kotlin CarritoViewModel.kt:43
private fun generarCodigoQRUnico(entradaId: Long?, email: String): String {
    val timestamp = System.currentTimeMillis()
    return "QR-${email.hashCode()}-${entradaId ?: 0}-$timestamp"
}
```

<Accordion title="QR Code Format">
  The QR code format ensures uniqueness by combining:
  - **Email Hash**: Hashed user email for privacy
  - **Ticket ID**: Unique ticket identifier
  - **Timestamp**: Purchase time in milliseconds
  
  Example: `QR-123456789-42-1678901234567`
</Accordion>

### Marking Tickets as Used

Workers can scan QR codes to mark tickets as used:

```kotlin CarritoViewModel.kt:159
fun marcarEntradaUsadaRemota(codigoQR: String) {
    viewModelScope.launch {
        if (!NetworkUtils.isNetworkAvailable(context())) {
            mensajeOperacion.value = "Sin conexion a internet"
            return@launch
        }

        try {
            val response = withContext(Dispatchers.IO) {
                RetrofitInstance.apiEntradas.marcarEntradaUsada(codigoQR)
            }
            if (response.isSuccessful) {
                val codigoLimpio = codigoQR.trim().uppercase()
                val actualizados = ticketsUsuario.map {
                    if (it.codigoQR?.uppercase() == codigoLimpio)
                        it.copy(estado = "ocupada")
                    else it
                }
                ticketsUsuario.clear()
                ticketsUsuario.addAll(actualizados)
                mensajeOperacion.value = "Entrada validada correctamente"
            }
        } catch (e: Exception) {
            mensajeOperacion.value = "Error de red: ${e.localizedMessage}"
        }
    }
}
```

## Ticket Management Operations

### Synchronize Tickets

Users can refresh their ticket list from the API:

```kotlin CarritoViewModel.kt:106
private suspend fun sincronizarEntradasDesdeApi(email: String) {
    if (!NetworkUtils.isNetworkAvailable(context())) {
        mensajeOperacion.value = "Sin conexión a internet"
        return
    }
    try {
        val repo = ComprasRepository()
        val response = withContext(Dispatchers.IO) { repo.obtenerCompras() }

        if (response.isSuccessful) {
            val todas = response.body() ?: emptyList()
            val comprasUsuario = todas.filter { it.usuario.email.equals(email, true) }
            val ticketsUsuarioActual = comprasUsuario.flatMap { 
                it.compraEntradas ?: emptyList() 
            }

            dataStore.saveEntradas(email, ticketsUsuarioActual.mapNotNull { it.entrada })
            ticketsUsuario.clear()
            ticketsUsuario.addAll(ticketsUsuarioActual)
            mensajeOperacion.value = "Entradas sincronizadas correctamente"
        }
    } catch (e: Exception) {
        mensajeOperacion.value = "Error de red: ${e.localizedMessage}"
    }
}
```

## Error Handling

The ticket system handles various error scenarios:

- **No Internet Connection**: "Sin conexion a internet"
- **API Errors**: "Error: [HTTP status code]"
- **Network Errors**: "Error de red"
- **Empty Results**: "No hay entradas disponibles"

<Note>
All ticket data is synchronized with the backend API to ensure consistency across devices and sessions.
</Note>
