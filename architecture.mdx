## Overview

RegistroX follows modern Android development best practices, implementing the **MVVM (Model-View-ViewModel)** architectural pattern with clean separation of concerns. The app is built with Jetpack Compose for UI, Kotlin Coroutines for asynchronous operations, and follows a layered architecture approach.

## Architecture Layers

The application is organized into distinct layers, each with specific responsibilities:

```
┌─────────────────────────────────────┐
│        Presentation Layer           │
│   (UI, Screens, ViewModels)         │
├─────────────────────────────────────┤
│       Navigation Layer              │
│   (Routes, NavGraph, NavItems)      │
├─────────────────────────────────────┤
│        Domain Layer                 │
│   (Business Logic, Use Cases)       │
├─────────────────────────────────────┤
│         Data Layer                  │
│ (Repositories, DataStore, Models)   │

│       Network Layer                 │
│   (Retrofit, API Services)          │
└─────────────────────────────────────┘
```

### ViewModel

ViewModels manage UI state and business logic, surviving configuration changes:

```kotlin
class LoginViewModel(
    application: Application,
    private val authRepository: AuthRepository,
    private val authDataStore: AuthDataStore
) : AndroidViewModel(application) {
    
    // UI State
    private val _formState = MutableStateFlow(LoginFormState())
    val formState: StateFlow<LoginFormState> = _formState
    
    private val _user = MutableStateFlow<User?>(null)
    val user: StateFlow<User?> = _user
    
    private val _isLoading = MutableStateFlow(false)
    val isLoading: StateFlow<Boolean> = _isLoading
    
    // Initialize from persisted data
    init {
        viewModelScope.launch {
            val email = authDataStore.email.first() ?: ""
            val role = authDataStore.role.first() ?: ""
            
            if (email.isNotBlank() && role.isNotBlank()) {
                _user.value = User(
                    id = 0L,
                    email = email,
                    role = if (role == "TRABAJADOR") Role.TRABAJADOR else Role.USUARIO
                )
            }
        }
    }
    
    // Business Logic
    fun login() {
        viewModelScope.launch {
            _isLoading.value = true
            
            try {
                val email = _formState.value.email.trim()
                val password = _formState.value.password
                
                val usuarioAPI = authRepository.login(email, password)
                
                if (usuarioAPI != null) {
                    val roleEnum = if (usuarioAPI.rol.id?.toInt() == 1) 
                        Role.TRABAJADOR else Role.USUARIO
                    
                    _user.value = User(
                        id = usuarioAPI.id ?: 0L,
                        email = usuarioAPI.email,
                        role = roleEnum
                    )
                    
                    authDataStore.saveUser(usuarioAPI.email, roleEnum.name)
                } else {
                    _formState.update { it.copy(loginError = "Credenciales incorrectas") }
                }
            } catch (e: Exception) {
                _formState.update { it.copy(loginError = "Error: ${e.localizedMessage}") }
            }
            
            _isLoading.value = false
        }
    }
}
```

**ViewModel Features:**
- **StateFlow** for reactive state management
- **viewModelScope** for lifecycle-aware coroutines
- **Separation of concerns** between UI and business logic
- **State preservation** across configuration changes

<Note>
ViewModels use `MutableStateFlow` internally for state updates, but expose immutable `StateFlow` to views, following the **unidirectional data flow** pattern.
</Note>

## Data Layer

### Repository Pattern

Repositories abstract data sources and provide clean APIs to ViewModels:

```kotlin
class AuthRepository {
    
    suspend fun register(email: String, password: String): Boolean {
        return try {
            val rol = if (email.endsWith("@registrox.cl"))
                Rol(id = 1, nombre = "TRABAJADOR")
            else
                Rol(id = 2, nombre = "USUARIO")
            
            val nuevoUsuario = Usuario(
                email = email,
                password = password,
                rol = rol
            )
            
            val response = withContext(Dispatchers.IO) {
                RetrofitInstance.api.createUsuario(nuevoUsuario)
            }
            
            response.isSuccessful
        } catch (e: Exception) {
            Log.e("API_REGISTER", "Error: ${e.localizedMessage}")
            false
        }
    }
    
    suspend fun login(email: String, password: String): Usuario? {
        return try {
            val response = withContext(Dispatchers.IO) {
                RetrofitInstance.api.getUsuarios()
            }
            
            if (response.isSuccessful) {
                val usuarios = response.body() ?: emptyList()
                usuarios.find { it.email.equals(email, ignoreCase = true) }
            } else null
        } catch (e: Exception) {
            Log.e("API_LOGIN", "Error: ${e.localizedMessage}")
            null
        }
    }
}
```

**Repository Responsibilities:**
- Abstract API calls from ViewModels
- Handle threading with coroutines
- Transform API responses to domain models
- Centralize error handling and logging

<Tip>
Repositories use `withContext(Dispatchers.IO)` to ensure network operations run on background threads, keeping the main thread responsive.
</Tip>

### DataStore

RegistroX uses Jetpack DataStore for persistent key-value storage:

```kotlin
val Context.authDataStore by preferencesDataStore("auth_prefs")

class AuthDataStore(private val context: Context) {
    
    companion object {
        private val EMAIL_KEY = stringPreferencesKey("email")
        private val ROLE_KEY = stringPreferencesKey("role")
    }
    
    val email: Flow<String?> = context.authDataStore.data.map { prefs ->
        prefs[EMAIL_KEY] ?: ""
    }
    
    val role: Flow<String?> = context.authDataStore.data.map { prefs ->
        prefs[ROLE_KEY] ?: ""
    }
    
    suspend fun saveUser(email: String, role: String) {
        context.authDataStore.edit { prefs ->
            prefs[EMAIL_KEY] = email
            prefs[ROLE_KEY] = role
        }
    }
}
```

**DataStore Advantages:**
- **Type-safe** with PreferencesKeys
- **Asynchronous** with Kotlin Flow
- **Coroutine-based** for suspend functions
- **Atomic transactions** for data consistency

## Network Layer

### Retrofit Configuration

The app uses Retrofit for RESTful API communication:

```kotlin
object RetrofitInstance {
    
    private const val BASE_URL = "https://api-rest-registrox.onrender.com/"
    
    private val client = OkHttpClient.Builder()
        .addInterceptor(HttpLoggingInterceptor().apply {
            level = HttpLoggingInterceptor.Level.BODY
        })
        .build()
    
    val apiEntradas: EntradasApiService by lazy {
        Retrofit.Builder()
            .baseUrl(BASE_URL)
            .addConverterFactory(GsonConverterFactory.create())
            .client(client)
            .build()
            .create(EntradasApiService::class.java)
    }
    
    val apiCompras: ComprasApiService by lazy {
        Retrofit.Builder()
            .baseUrl(BASE_URL)
            .addConverterFactory(GsonConverterFactory.create())
            .client(client)
            .build()
            .create(ComprasApiService::class.java)
    }
    
    val api: UsuarioApiService by lazy {
        Retrofit.Builder()
            .baseUrl(BASE_URL)
            .addConverterFactory(GsonConverterFactory.create())
            .client(client)
            .build()
            .create(UsuarioApiService::class.java)
    }
}
```

**Network Features:**
- **Lazy initialization** for efficient resource usage
- **HTTP logging** for debugging
- **Gson serialization** for JSON parsing
- **Service separation** for API endpoint organization

## Navigation Architecture

### Navigation Graph

The app uses Jetpack Navigation Compose for declarative navigation:

```kotlin
@Composable
fun NavGraph(
    navController: NavHostController,
    carritoViewModel: CarritoViewModel,
    loginViewModel: LoginViewModel,
    registerViewModel: RegisterViewModel,
    profileViewModel: ProfileViewModel,
    entradasApiViewModel: EntradasApiViewModel,
    comprasViewModel: ComprasViewModel,
    modifier: Modifier = Modifier
) {
    val user by loginViewModel.user.collectAsStateWithLifecycle()
    
    val startDestination = if (user == null) Routes.LOGIN else Routes.HOME
    
    NavHost(
        navController = navController,
        startDestination = startDestination,
        modifier = modifier
    ) {
        composable(Routes.LOGIN) {
            LoginScreen(navController, loginViewModel)
        }
        
        composable(Routes.HOME) {
            HomeScreen(navController, carritoViewModel)
        }
        
        composable(Routes.ENTRADAS) {
            EntradasScreen(navController = navController, carritoViewModel = carritoViewModel)
        }
        
        composable("${Routes.DETALLE}/{codigoQR}") { backStackEntry ->
            val codigoQR = backStackEntry.arguments?.getString("codigoQR") ?: ""
            DetalleEntradaScreen(navController, codigoQR)
        }
        
        composable(Routes.TRABAJADOR) {
            val current = user
            if (current?.role == Role.TRABAJADOR)
                HomeTrabajadorScreen(
                    onBackClick = { navController.navigate(Routes.HOME) },
                    carritoViewModel = carritoViewModel
                )
        }
    }
}
```

**Navigation Features:**
- **Type-safe routes** via constants
- **Dynamic start destination** based on auth state
- **Role-based routing** with conditional screens
- **Deep linking support** with parameterized routes

## State Management

### Unidirectional Data Flow

RegistroX follows the unidirectional data flow pattern:

```
┌──────────────────────────────┐
│         UI Events            │
│  (Button clicks, input)      │
└──────────┬───────────────────┘
           │
           ▼
┌──────────────────────────────┐
│        ViewModel             │
│  (Process events, update)    │
└──────────┬───────────────────┘
           │
           ▼
┌──────────────────────────────┐
│       Repository             │
│   (Fetch/update data)        │
└──────────┬───────────────────┘
           │
           ▼
┌──────────────────────────────┐
│      State Updates           │
│   (via StateFlow/Flow)       │
└──────────┬───────────────────┘
           │
           ▼
┌──────────────────────────────┐
│      UI Recomposition        │
│   (Automatic updates)        │
└──────────────────────────────┘
```

### State Management in Practice

```kotlin
class CarritoViewModel(
    application: Application,
    private val dataStore: EntradasDataStore,
    private val authDataStore: AuthDataStore? = null
) : AndroidViewModel(application) {
    
    // Mutable state (internal)
    val carrito = mutableStateListOf<Entrada>()
    val ticketsUsuario = mutableStateListOf<CompraEntrada>()
    val mensajeOperacion = mutableStateOf("")
    val isLoading = mutableStateOf(false)
    
    // Event handlers
    fun agregar(entrada: Entrada) {
        val existente = carrito.find { it.id == entrada.id }
        if (existente != null) {
            val actualizado = existente.copy(cantidad = existente.cantidad + entrada.cantidad)
            carrito.remove(existente)
            carrito.add(actualizado)
        } else carrito.add(entrada)
        mensajeOperacion.value = "Entrada agregada al carrito"
    }
    
    fun comprar() {
        viewModelScope.launch {
            if (carrito.isEmpty()) {
                mensajeOperacion.value = "No hay entradas en el carrito"
                return@launch
            }
            
            // Business logic...
            val response = withContext(Dispatchers.IO) {
                RetrofitInstance.apiCompras.createCompra(compra)
            }
            
            if (response.isSuccessful) {
                mensajeOperacion.value = "Compra registrada correctamente"
                limpiar()
                sincronizarEntradasDesdeApi(email)
            }
        }
    }
}
```

<Tip>
Using `mutableStateListOf` and `mutableStateOf` from Compose enables automatic UI recomposition when state changes, eliminating manual UI update code.
</Tip>

## Error Handling

### Layered Error Management

Errors are handled at multiple layers:

1. **Network Layer**: Catch HTTP errors and exceptions
2. **Repository Layer**: Log errors and return null/false indicators
3. **ViewModel Layer**: Update UI state with error messages
4. **UI Layer**: Display error messages to users

```kotlin
// Repository layer
suspend fun login(email: String, password: String): Usuario? {
    return try {
        val response = withContext(Dispatchers.IO) {
            RetrofitInstance.api.getUsuarios()
        }
        
        if (response.isSuccessful) {
            // Success path
            response.body()?.find { it.email.equals(email, ignoreCase = true) }
        } else {
            Log.e("API_LOGIN", "Error HTTP ${response.code()} ${response.message()}")
            null
        }
    } catch (e: Exception) {
        Log.e("API_LOGIN", "Error: ${e.localizedMessage}")
        null
    }
}

// ViewModel layer
fun comprar() {
    viewModelScope.launch {
        try {
            // Business logic
            val response = withContext(Dispatchers.IO) {
                RetrofitInstance.apiCompras.createCompra(compra)
            }
            
            if (response.isSuccessful) {
                mensajeOperacion.value = "Compra registrada correctamente"
            } else {
                mensajeOperacion.value = "Error al registrar compra (${response.code()})"
            }
        } catch (e: Exception) {
            mensajeOperacion.value = "Error de red: ${e.localizedMessage}"
        }
    }
}
```
