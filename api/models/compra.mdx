## Compra

The `Compra` model represents a purchase transaction in the RegistroX application. It contains information about the purchase date, the user who made the purchase, and all the ticket instances included in the transaction.

```kotlin
data class Compra(
    val id: Long? = null,
    val fechaCompra: String? = null,
    val usuario: Usuario,
    val compraEntradas: List<CompraEntrada>? = emptyList()
)
```

### Fields

<ResponseField name="id" type="Long">
  Unique identifier for the purchase transaction (nullable for new purchases)
</ResponseField>

<ResponseField name="fechaCompra" type="String">
  Date and time when the purchase was made (typically in ISO 8601 format)
</ResponseField>

<ResponseField name="usuario" type="Usuario" required>
  The user who made the purchase. See [Usuario](/api/models/user#usuario) model
</ResponseField>

<ResponseField name="compraEntradas" type="List<CompraEntrada>" default="emptyList()">
  List of ticket instances included in this purchase. Each entry represents a specific ticket with its own QR code and state
</ResponseField>

## CompraEntrada

The `CompraEntrada` model represents an individual ticket instance within a purchase. This model links a purchase to a specific ticket entry and includes its own QR code and status tracking.

```kotlin
data class CompraEntrada(
    val id: Long? = null,
    val estado: String = "disponible",
    val codigoQR: String? = null,
    val entrada: Entrada? = null
)
```

### Fields

<ResponseField name="id" type="Long">
  Unique identifier for this ticket instance (nullable for new instances)
</ResponseField>

<ResponseField name="estado" type="String" default="disponible">
  Current status of this specific ticket instance (e.g., "disponible", "usado", "cancelado")
</ResponseField>

<ResponseField name="codigoQR" type="String">
  Unique QR code for this specific ticket instance used for validation (nullable until generated)
</ResponseField>

<ResponseField name="entrada" type="Entrada">
  Reference to the ticket entry this instance is based on. See [Entrada](/api/models/entrada) model
</ResponseField>

## Purchase Flow

The typical purchase flow involves these models:

1. **Selection**: User browses available `Entrada` tickets
2. **Purchase**: A `Compra` is created with the current user
3. **Instance Creation**: For each ticket in the cart, a `CompraEntrada` is created with a unique QR code
4. **Validation**: Workers scan the QR code to validate and mark the `CompraEntrada` as used

## Ticket Instance States

Each `CompraEntrada` tracks its own state independently:

| Estado | Description |
|--------|-------------|
| `disponible` | Ticket instance is ready to be used |
| `usado` | Ticket instance has been scanned and used for entry |
| `cancelado` | Ticket instance has been cancelled or refunded |
